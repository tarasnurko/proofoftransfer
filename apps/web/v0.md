proof of transfer app - is web3 app where user can prove that they sent tokens to some specific address. It is build for evm chains and use etherscan as api to retrieve erc20 transfers for some specific address (reciever). There is 3 main flows. 1 - user create "claim" with some constraints which any user can generate proof for. For example constraints "Have you transfered at least x and at most y tokens to recipient Alice in previous week". It should be a form. Second flow is that any user can go to this claim, and generate poof that they in fact did transfer - it fetches all transfers which corresponds to claim constraints, then generate merkle proof for them all, and then pass all this data with actual prover transfers to circuit to generate proof and verify. Third flow is that any user can go to claim and see all proofs
